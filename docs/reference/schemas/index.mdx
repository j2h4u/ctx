description: "Versioned JSON schemas for ctx Work compatibility records, transcripts, events, tool calls, and plugins"

The public product noun is Work. The current v1 schema compatibility surface
still uses `agent-work` names for files and types, so records are exposed as
`AgentWork`, `ChangeSet`, `Contribution`, and `ContributionEndpoint`.

Treat `agent-work` as compatibility naming for this branch, not as a second
public vocabulary. Hosted/team service internals are not part of these schema
names.

## Schemas

- [`schemas/agent-work/v1.schema.json`](../../../schemas/agent-work/v1.schema.json):
  `AgentWork`, the workspace aggregate with `change_sets` and `contributions`.
- [`schemas/agent-work/change-set.v1.schema.json`](../../../schemas/agent-work/change-set.v1.schema.json):
  `ChangeSet`.
- [`schemas/agent-work/contribution.v1.schema.json`](../../../schemas/agent-work/contribution.v1.schema.json):
  `Contribution`.
- [`schemas/events/v1.schema.json`](../../../schemas/events/v1.schema.json):
  `EventEnvelope`, matching the session event wire shape.
- [`schemas/events/tool-call.v1.schema.json`](../../../schemas/events/tool-call.v1.schema.json):
  `ToolCall`, matching the persisted session turn tool record.
- [`schemas/transcripts/v1.schema.json`](../../../schemas/transcripts/v1.schema.json):
  `TranscriptRecord`, a tagged union for messages, event envelopes, tool calls,
  and artifacts.
- [`schemas/plugins/plugin-manifest.v1.schema.json`](../../../schemas/plugins/plugin-manifest.v1.schema.json):
  `PluginManifest`.

## Versioning

These schemas describe public v1 wire shapes. Records that carry
`schema_version` currently use `1`, matching the Rust model defaults for Work
compatibility records and plugin manifests. `ChangeSet` and `Contribution` may
also carry `source_records` entries that preserve source record hashes for
imported or projected Work.

The schemas include fields that are useful for local provenance, such as
`repo_root` and artifact `absolute_path`. Shareable export or sync producers
should omit or redact host-local paths unless the user explicitly opts into
preserving them.

Source record hashes are useful for detecting changed imported records and
preserving provenance across import/export. They are not a hosted, tamper-proof
audit guarantee by themselves.

## Compatibility

`ContributionEndpoint` uses canonical snake_case kind names such as
`change_set`, `pull_request`, and `review_attestation`. The schema also accepts
the kebab-case endpoint aliases that the Rust deserializers accept for imported
records.
